## Реализовано

### Алгоритмы

В проекте реализованы базовые и продвинутые методы гладкой безусловной оптимизации:

- `LineSearchTool`
- `gradient_descent(...)`
- `newton(...)`
- `linear_conjugate_gradients(...)`
- `nonlinear_conjugate_gradients(...)`
- `hessian_free_newton(...)`

### Стратегии выбора шага

Для подбора длины шага используются:

- `Constant`

Для исследовательского трека отдельно сравниваются:

- классический `L-BFGS + Wolfe`;
- `Cautious L-BFGS + Armijo`.

- `QuadraticOracle`
- `DiagonalQuadraticOracle`
- `BealeOracle`
- `PoissonL2Oracle`
- `LogisticL2Oracle`

Дополнительно реализованы:

- `hess_vec(x, v)` для ML-оракулов без явного построения гессиана;
- `hess_vec_finite_diff(...)` для разностной проверки произведения гессиана на вектор.

## Содержание ноутбука и отчёта

### 1. Проверка `hess_vec`

В ноутбуке сначала проверяется корректность произведения гессиана на вектор через сравнение:

- аналитического `hess_vec`;
- разностной аппроксимации `hess_vec_finite_diff`.

### 2. Эксперимент 2.2

Исследуется зависимость числа итераций линейного метода сопряженных градиентов от:

- числа обусловленности;
- размерности пространства.

Также выполняется сравнение с градиентным спуском на квадратичных задачах.

### 3. Эксперимент 2.3

Исследуется влияние размера памяти `L` в L-BFGS:

- график по номеру итерации;
- график по реальному времени;
- график итогового времени от `L`.

По ТЗ случай `L = 0` трактуется как градиентный спуск с Вульфом.

### 4. Эксперимент 2.4

На реальных ML-задачах сравниваются:

- `Nonlinear CG`
- `Hessian-Free Newton`
- `L-BFGS`
- `Gradient Descent`
- `Newton`

По заданию на один график нужно добавить также методы из первой лабораторной работы — GD и полный Newton.

### 5. Эксперимент 2.5

Выполняется микропрофилирование затрат по трём частям:

- оракул;
- линейная алгебра метода;
- линейный поиск.

### 6. Эксперимент 2.6

Исследуется связь между:

- `f_train(x_k)`,
- `||∇f(x_k)||`,
- качеством на тестовой выборке.

Для классификации используется Accuracy, для регрессии — MSE / MAE.

### 7. Исследовательский трек 4

Сравниваются:

- классический `L-BFGS + Wolfe`;
- `Cautious L-BFGS + Armijo`.

В треке анализируются:

- устойчивость без Вульфа;
- число отброшенных пар `(s_k, y_k)`;
- среднее число вызовов оракула на итерацию;
- реальное время работы. fileciteturn5file0

## Краткие выводы по работе

- Линейный CG существенно лучше GD на квадратичных задачах с плохой обусловленностью.
- L-BFGS даёт хороший баланс между стоимостью одной итерации и скоростью сходимости.
- Hessian-Free Newton часто выигрывает по числу внешних итераций, но не всегда по реальному времени.
- На широких разреженных данных стоимость работы с кривизной становится критичной.
- В треке 4 отказ от Вульфа может быть выгоден по времени только если “осторожное” обновление истории не слишком часто отбрасывает пары и сохраняет стабильность метода.
